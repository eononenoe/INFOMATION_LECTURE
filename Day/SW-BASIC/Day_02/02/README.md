# 🔗 데이터 링크 계층 (Data Link Layer)

---

## 📌 개요 (Overview)

**데이터 링크 계층은 OSI 7계층 중 2계층으로, 네트워크에서 "누가 누구에게 보내는지"와 "정확하게 잘 도착했는지"를 책임지는 계층이에요.**  
비유하자면, 단순히 전기 신호만 보내는 물리계층 위에 있는, **'주소 확인, 오류 검사, 흐름 조절'을 해주는 택배 기사**라고 할 수 있어요! 🧑‍💼📦

---

## 💡 핵심 개념 요약 (Key Concepts)

### 🚚 역할
- **장치 간 Point-to-Point(점대점) 연결** 방식으로 데이터 운송 결정
- 데이터를 **프레임(Frame)** 단위로 나누어 전송

### 🛠 데이터 링크 계층의 필수 요소
- **주소 지정**: 상대방 장치가 누군지 알아야 함 (MAC 주소)
- **오류 제어**: 데이터가 손상되었는지 확인
- **흐름 제어**: 빠르게 보내는 쪽 vs 느리게 받는 쪽 속도 조절

### 🖧 장치
- **L2 스위치**  
  - 각 장치의 **MAC 주소를 기억**하고, 필요한 장치에만 데이터를 전달
  - **충돌(Collision)**을 방지하여 효율적인 통신 제공

### 📡 대표 프로토콜
- **LAN** (빠른 속도 중점):
  - **이더넷(Ethernet)**: 초창기에는 **CSMA/CD** 방식 사용
- **WAN** (신뢰성 중점):
  - **HDLC**: 흐름 제어 + 오류 제어 포함
  - **PPP** 등도 포함됨

---

## ⚠ 주의사항 (Cautions)

- **이더넷의 초기 문제점**:  
  - **CSMA/CD**는 데이터 충돌 시 모든 장비 통신 중단됨 (Jamming Time 발생)
- **해결 방법**:  
  - **전이중(Full Duplex) + L2 스위치 사용** → 충돌 없음!

---

## 🧪 예제 또는 비유 (Examples or Analogies)

### ✉️ 택배 배송 비유
- **데이터 링크 계층**은 택배 기사!
  - **주소 확인** (MAC 주소)
  - **부서졌는지 검사** (오류 제어)
  - **너무 많이 안 보내기** (흐름 제어)

### 🕹 게임 서버 예시
- 너무 많은 유저가 동시에 접속하면 **서버가 멈추는 현상(Collision)** 발생  
  → 그래서 요즘은 **유저당 분산 서버(L2 Switch)** 방식으로 개선!

---

## ✅ 한 줄 요약 (1-Line Summary)

> **데이터 링크 계층은 '누구에게 정확하게 잘 보내는지'를 책임지는 네트워크의 똑똑한 택배 기사이다! 📦🔍**

# 🌐 네트워크 계층 (Network Layer)

---

## 📌 개요 (Overview)

**네트워크 계층은 OSI 7계층 중 3계층이며, 데이터를 "어떻게 목적지까지 보낼지 경로를 찾는 역할"을 해요.**  
데이터가 단순히 옆 사람에게만 가는 게 아니라, **멀리 떨어진 상대방에게까지** 가야 할 때, 이 계층이 **길을 안내하는 네트워크 내비게이션**이 되어줘요! 🚗🗺️

---

## 💡 핵심 개념 요약 (Key Concepts)

### 🧭 역할
- **경로 탐색(Routing)**: 데이터를 목적지까지 보내기 위한 최적 경로 선택
- **논리적 주소 부여(IP 주소)**

### 🛠 장치
- **라우터(Router)**: 네트워크 사이의 길을 안내해주는 길잡이
- **L3 스위치**: 라우팅 기능이 있는 스위치 (빠른 처리 + 경로 설정)

### 🔐 주요 프로토콜
- **IP (Internet Protocol)**  
  - 장치를 구분하기 위한 **논리적 주소 부여**
  - IPv4 / IPv6 형태

- **ICMP (Internet Control Message Protocol)**  
  - 통신이 가능한지 확인 (ex: 핑 테스트 🏓)

- **ARP (Address Resolution Protocol)**  
  - **IP 주소 → MAC 주소**로 변환해주는 친구

- **Routing Protocols**  
  - **네트워크용 내비게이션**  
  - 예: **RIP, Static Routing**

---

## ⚠ 주의사항 (Cautions)

- **IP는 실제 주소(물리주소)가 아니라 논리주소**예요!
- **라우터는 서로 다른 네트워크 간의 경로만 연결 가능** (같은 네트워크 안의 연결은 스위치가 담당)
- **ICMP는 데이터를 보내는 게 아니라, '보낼 수 있는지'를 체크하는 프로토콜**

---

## 🧪 예제 또는 비유 (Examples or Analogies)

### 🛰️ 택배 배송 비유 (고속도로 버전)
- 데이터는 **택배 상자**
- 네트워크 계층은 **배송 경로를 짜주는 내비게이션**
- IP = 택배 주소
- ICMP = "그 주소 배송 가능한가요?" 전화
- ARP = "주소는 있는데, 정확한 집 위치는 어디야?" 묻는 일

---

## ✅ 한 줄 요약 (1-Line Summary)

> **네트워크 계층은 데이터를 목적지까지 보내기 위해 길을 찾아주는 네트워크의 내비게이션이다! 🧭🚗**

# 🚚 전송 계층 (Transport Layer)

---

## 📌 개요 (Overview)

**전송 계층은 OSI 7계층 중 4계층으로, 송신자와 수신자 간에 '신뢰성 있는 데이터 전달'을 책임지는 계층입니다.**  
네트워크가 복잡하더라도 **양 끝단(End-to-End)** 사이에서 **데이터가 잘 전달되었는지 확인**하고, 문제가 생기면 재전송까지 처리해주는 **꼼꼼한 택배 관리자** 같은 역할이에요! 🧑‍💼📦✅

---

## 💡 핵심 개념 요약 (Key Concepts)

### 🧭 역할
- 송수신 장치 간의 **신뢰성 있는 데이터 전송 보장**
- **데이터 흐름 제어**, **오류 복구**, **세그먼트 분할/조립**

### 🛠 장치
- **L4 스위치**  
  - TCP/UDP 포트 번호까지 보고 처리 가능한 스위치  
  - **트래픽 분산, 로드 밸런싱**에 활용

---

### 🔐 주요 프로토콜

#### 🧱 TCP (Transmission Control Protocol)
- **연결 지향형** 프로토콜  
- 데이터 전송 전 **3-way handshaking**으로 연결 설정  
- 데이터 전송 후 **수신 여부 확인 + 재전송 가능**
- 특징:
  - **신뢰성 O**
  - **속도 느림** (확인하고 또 확인)

#### 🪂 UDP (User Datagram Protocol)
- **비연결 지향형** 프로토콜
- **빠르게 보내고 끝!**  
- 실시간 게임, 영상 스트리밍, VoIP 등에 사용됨
- 특징:
  - **신뢰성 X**
  - **속도 빠름**
  - **멀티캐스팅 가능**

---

## ⚠ 주의사항 (Cautions)

- **TCP는 신뢰성은 높지만 속도가 느릴 수 있음**  
- **UDP는 빠르지만 데이터 손실에 대한 보장이 없음**
- **UDP는 중요한 데이터를 보낼 때는 부적합할 수 있음**

---

## 🧪 예제 또는 비유 (Examples or Analogies)

### 📦 택배 비유
- **TCP** = 등기 택배  
  - 보내기 전 전화, 도착하면 확인 문자, 분실 시 보상  
- **UDP** = 일반우편  
  - 그냥 보내고 끝! 빨리 가지만 분실 시 책임 없음

### 🎮 실생활 사용 예시
- **TCP 사용 서비스**: 웹사이트 접속(HTTP/HTTPS), 이메일(SMTP/IMAP)
- **UDP 사용 서비스**: 유튜브 실시간 스트리밍, 온라인 게임, 화상 회의

---

## ✅ 한 줄 요약 (1-Line Summary)

> **전송 계층은 송수신 간 데이터가 정확히 전달되도록 책임지는 꼼꼼한 배달 관리자! 🚚📬**

# 🧱 OSI 7계층 (Application ~ Physical)

---

## 📌 개요 (Overview)

**OSI 7계층 모델(Open Systems Interconnection Model)**은 네트워크 통신을 7단계로 나눈 개념으로, 각 계층은 특정한 역할을 맡아 데이터를 송수신합니다.  
쉽게 말해, 데이터가 내 컴퓨터에서 상대방 컴퓨터로 갈 때까지 **“어떻게 처리되고 전달되는지”**를 단계별로 설명해주는 설계도입니다! 🏗️

---

## 💡 핵심 계층 요약 (Key Concepts)

---

### 🔟 7계층: 응용 계층 (Application Layer)
- **사용자가 직접 만나는 계층**
- 웹 브라우저, 이메일, FTP, 메신저 등
- 실제 서비스와 앱이 동작하는 위치
- 💡 *"사람이 직접 사용하는 서비스!"*

---

### 🖼️ 6계층: 표현 계층 (Presentation Layer)
- 데이터의 **형식/양식 변환**
- 예: **암호화, 복호화, 압축, 이미지 포맷 변환 등**
- 💡 *"서로 알아들을 수 있게 번역!"*

---

### 🧷 5계층: 세션 계층 (Session Layer)
- **통신 세션(연결)**을 만들고 유지하다가 종료
- 예: 로그인 상태 유지, 화상회의 연결 유지
- 💡 *"통신의 입장-퇴장 도우미!"*

---

### 🚚 4계층: 전송 계층 (Transport Layer)
- **송신자-수신자 간 신뢰성 있는 데이터 전송**
- 주요 프로토콜: **TCP(신뢰 O), UDP(속도 O)**
- 장치: **L4 Switch (로드 밸런서)**

---

### 🌐 3계층: 네트워크 계층 (Network Layer)
- **데이터가 갈 길(경로) 탐색**
- 주요 프로토콜: **IP, ICMP, ARP**
- 장치: **라우터, L3 스위치**

---

### 🔗 2계층: 데이터 링크 계층 (Data Link Layer)
- **직접 연결된 장치 간 데이터 전송 + 오류/흐름 제어**
- 주요 프로토콜: **Ethernet, HDLC**
- 장치: **L2 스위치**

---

### ⚡ 1계층: 물리 계층 (Physical Layer)
- **전기 신호로 데이터 전달 (하드웨어 수준)**
- 케이블, 허브, 리피터 등
- 💡 *"0과 1을 실제 전기 신호로 바꿔주는 층!"*

---

| 6    | 표현 계층        | 데이터 형식 변환 (암호화/압축)         | 이미지, 영상, 인코딩             |

| 2    | 데이터 링크 계층 | 장치 간 데이터 전송 + 오류제어         | L2 스위치, Ethernet, HDLC        |

## 🧪 예제 또는 비유 (Examples or Analogies)

### 📦 택배 시스템 비유
- 응용 계층: 고객이 택배 요청
- 표현 계층: 주소 포맷, 문자 인코딩

- 전송 계층: 택배 추적 + 도착 확인

- 데이터 링크 계층: 중간 물류 센터 전송
- 물리 계층: 실제 택배 차량, 도로
